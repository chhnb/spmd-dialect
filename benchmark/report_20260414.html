Taichi 在 207K 上最快（~12% faster than Graph）。nsys profiling 确认原因：Taichi JIT 生成了更高效的 CUDA kernel。
calculate_flux: CUDA 46 regs/thread, block=256 → Taichi 32 regs/thread, block=128。
Taichi 的低 register 使用让 occupancy 提高 3.2×（16 vs 5 blocks/SM），更好地隐藏内存延迟。
这不是 launch overhead 差异，是 JIT 编译器优化质量的差异。CUDA nvcc 对 OSHER solver 的 register 分配不如 Taichi LLVM backend 高效。

Key Findings

1. Launch Overhead 是真实的，但 size-dependent

Case小网格 overhead大网格 overhead
HydroF2 (2 kern/step)62% (15.87 us)22% (16.11 us)
HydroF1 (1 kern/step)24% (7.51 us)6% (7.96 us)

绝对值 ~8-16 us 不变（取决于 kernels/step），但占比随 GPU compute 增大而下降。

2. CUDA Graph 是最优的 launch 策略

在所有小网格 case 上 Graph 最快。大网格上 Graph ≈ Async ≈ Taichi（GPU compute 主导）。

3. Device-side Graph Launch (Blink 方式) 全面不如 Host Graph

每步恒定 +7 us overhead。原因：GPU command processor 从 device 端处理 graph launch 比 host 端慢。

4. Persistent (Megakernel) 不是 launch 策略

Persistent 改了 kernel 代码（把 host 编排搬进 kernel + grid.sync()）。
在 F1 207K 上 Persistent(128) 反而比 Graph(120) 慢——说明 megakernel 的 occupancy 损失可能抵消 launch overhead 节省。
Persistent 的收益混合了 launch elimination + L2 locality + occupancy 变化，不可简单归因于 launch overhead。

5. DSL Dispatch Overhead ≈ 25-50 us/kernel (Taichi), ~10 us/kernel (Warp)

源码验证：Taichi 和 Warp 都是 async dispatch（不 sync）。
小网格上 dispatch 是瓶颈（Taichi 5x 慢于 Graph）。
大网格上 dispatch 被 GPU compute overlap → Taichi 接近甚至略快于 CUDA Graph。

Related Work

PERKS / EBISU (Zhang, Wahib et al., ICS'23)

做了什么：Persistent kernel + temporal blocking for stencil。利用 GPU 的剩余 register 和 shared memory 缓存多个时间步数据，减少全局内存访问。

核心 insight：GPU kernel 通常不用完所有 on-chip 资源，剩余的可以用来做 temporal caching。低 occupancy + 高 temporal depth 是 Pareto 最优的。

和我们的关系：不在同一层对比。PERKS 是 kernel 级算法优化（temporal blocking），我们是 launch 级 characterization。PERKS 只适用于规则 stencil，我们覆盖通用 simulation（包括 CFD、粒子等 PERKS 无法处理的 case）。

Blink (arXiv'25)

做了什么：CPU-free LLM inference，用 device-side graph launch 让整个推理过程不回 host。

我们的测试结果：在 simulation 场景下 device-side graph launch 全面不如 host graph（每步 +7 us overhead）。原因：simulation kernel 小（几十 us），7 us overhead 占比大。LLM kernel 大（几 ms），7 us 可忽略。

PyGraph (arXiv'25)

CUDA Graph 的 selective deployment + cost-benefit analysis。我们的 cost model / strategy selector 可借鉴其方法论。
